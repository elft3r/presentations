The 7 Rs

Retire

  • Decommission applications that are obsolete
  • Example:
    • An old, custom-built inventory management system, which got replaced by a new ERP system
    • Freeing up resources and reducing maintenance costs

Retain

  • Keep the application on-premise
  • Application is not yet ready
  • Example:
    • A mainframe system that runs critical financial processing, that is challenging to migrate
    • Retain in the current environment and focus migrating less critical systems

Rehost

  • Lift and Shift
  • Example:
    • Quickly migrate an e-commerce application to the cloud
    • Don't change the core architecture, and quickly gain cloud benefits without major redevelopment

Replatform

  • Minor optimizations to the application
  • Example:
    • A customer management application with an on-premise database that's compatible with cloud services
    • Migrate to a managed cloud database service (e.g. Amazon RDS) to reduce operational overhead

Refactor or Re-Architect

  • Redesign applications for cloud-native benefits
  • Example:
    • Legacy application that cannot scale to meet the demand
    • Refactor by breaking it into microservices and following cloud-native principles

Repurchase

  • Replace existing application with cloud-based SaaS alternative
  • Example:
    • A company is running an on-premise version control system but wants to leverage a SaaS solution
    • Replace the current version control system with a SaaS solution like GitHub or GitLab

Relocate

  • Move entire virtualized environments without altering the platform
  • Example:
    • A company is running Hyper-V based applications on-premise and wants to leverage cloud flexibility without refactoring
    • Relocate to a Hyper-V solution in the cloud without modifying the VMs
