Migration Strategies

What is cloud migration?

Cloud migration is the process of moving a company's digital assets, services, databases, IT resources, and applications either partially, or wholly, into the cloud. Cloud migration is also about moving from one cloud to another.

The 7 Rs

The 7 Rs — Low Effort

Retire

Decommission obsolete apps

e.g. old inventory system replaced by ERP

Retain

Keep on-premises — not ready to migrate

e.g. mainframe running critical processing

Rehost

Lift & Shift — no architecture changes

e.g. move e-commerce app to cloud as-is

Relocate

Move VMs to cloud without modification

e.g. Hyper-V workloads to cloud as-is

About the migration strategies

The 7 Rs — Higher Effort

Replatform

Minor tweaks, use managed services

e.g. migrate on-prem DB to Amazon RDS

Refactor

Redesign for cloud-native benefits

e.g. break monolith into microservices

Repurchase

Replace with a SaaS alternative

e.g. on-prem VCS to GitHub or GitLab

About the migration strategies

Migration Strategy Takeaways

  • Not every application needs the same migration strategy
  • Start with low-effort strategies (Rehost, Retain) for quick wins
  • Refactor only when there is a clear business benefit
  • Evaluate SaaS alternatives before rebuilding
