Container History

What are containers?

A lightweight, isolated environment to run an application —
using features already built into the Linux kernel

"chroot on steroids"

— Jérôme Petazzoni

chroot

Isolate the filesystem

cgroups

Limit resources (CPU, memory, I/O)

namespaces

Virtualize resources (PID, network, users)

copy-on-write

Efficient layered filesystem

⚠️ Containers are OS-specific — a Windows container won’t run on Linux and vice versa

What are containers?

chroot — Where It All Started

  • Unix V7, 1979 — changes the apparent root directory for a process
  • The process sees its own /bin, /etc, /usr, … — first form of filesystem isolation
  • Limitations: no process isolation, no resource limits, root can escape
